// Installation Guide

How to Install

All mods use BepInEx. Takes about five minutes from scratch. No modded launcher needed — just drag and drop.

Install BepInEx
Grab BepInEx 6 from the official repo. Extract it into your Gorilla Tag game folder next to GorillaTag.exe.
Run the Game Once
Launch Gorilla Tag normally. BepInEx will generate its folder structure including the BepInEx/plugins/ folder. Close the game.
Drop in the DLL
Download the mod .dll from GitHub Releases. Paste it into BepInEx/plugins/.
Launch and Enjoy
Start the game. The mod loads automatically. Check BepInEx/LogOutput.log if something looks wrong.
$ cd "C:\Program Files (x86)\Steam\steamapps\common\Gorilla Tag"
$ dir BepInEx/plugins/
GhostHandCopy.dll
NeonTrails.dll
# both mods loaded and ready

// FAQ

Frequently Asked

What is a copy mod? +
A copy mod replicates a cosmetic or game element client-side — only you see the change. It doesn't modify game servers or other players' clients. Think of it as a local texture or asset override.
Will I get banned? +
Copy mods are client-side only and don't give gameplay advantages. However, Gorilla Tag's terms of service prohibit unauthorized modifications. Use in private lobbies with consenting players to stay safe. We take no responsibility for account actions.
Do these work on Quest? +
Some mods are Quest-compatible via sideloading tools like SideQuest. Each mod card is clearly labeled PC Only or Quest Compatible. Quest support is more limited because BepInEx isn't available on Android builds.
How do I contribute a mod? +
Fork the repo on GitHub, add your mod in its own folder with a README, and open a Pull Request. The maintainers review all submissions for malicious code before merging. Read CONTRIBUTING.md for the full guide.
My mod isn't loading — what do I check? +
First open BepInEx/LogOutput.log and search for errors. Most common issues: wrong BepInEx version (need 6.x for most mods here), .dll placed in the wrong folder, or a game update that changed internal class names. Open a GitHub issue with your log snippet.
